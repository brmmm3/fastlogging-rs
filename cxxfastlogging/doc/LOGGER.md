# `Logger` — Per-Domain Handle

`Logger` is a lightweight handle that can be registered with a `Logging`
instance and used from any thread without holding the main `Logging` object.
Multiple `Logger` instances share one `Logging` background thread.
`Logger` is an opaque Rust type held through `rust::Box<Logger>`.

## Constructors

### `Logger::new`

```cpp
static rust::Box<Logger> new(uint8_t level, rust::Str domain);
```

Creates a logger with the given filter level and domain label.  Thread-name
and thread-id logging are disabled.

> **Note:** In C++ code generated by the cxx bridge, the associated function
> `Logger::new` is callable as `Logger::new(level, domain)` — `new` is allowed
> here because it is a *static member function* name generated by cxx, not the
> C++ `new` operator expression.

### `Logger::new_ext`

```cpp
static rust::Box<Logger> new_ext(uint8_t level, rust::Str domain,
                                  bool tname, bool tid);
```

Extended constructor that optionally includes the calling thread's name and/or
id in every message.

## Registration

A `Logger` is inert until registered.  Registration connects its internal
channel to the parent `Logging` thread:

```cpp
// Register
logging->add_logger(*logger);

// Later — unregister
logging->remove_logger(*logger);
```

After `add_logger` the logger forwards messages to the shared background
thread.  After `remove_logger` (or after the `Logging` is dropped) log calls
will throw `rust::Error`.

## Configuration

```cpp
void set_level(uint8_t level);
uint8_t level() const;
void set_domain(rust::Str domain);
```

## Logging Methods

```cpp
int trace(rust::Str message);
int debug(rust::Str message);
int info(rust::Str message);
int success(rust::Str message);
int warning(rust::Str message);
int error(rust::Str message);
int critical(rust::Str message);
int fatal(rust::Str message);
int exception(rust::Str message);
```

Messages are tagged with the `Logger`'s own `domain`, independent of the
parent `Logging` domain.

## Thread Safety

`rust::Box<Logger>` is owned by one thread at a time.  The recommended pattern
for multi-threaded use is to create and register the logger in the main thread
*before* spawning, then move the box into the worker:

```cpp
#include "cxxfastlogging/h/fastlogging.h"
#include <thread>

int main() {
    rust::Vec<rust::Box<WriterConfig>> configs;
    configs.push_back(WriterConfig::new_console(DEBUG, true));
    auto logging = Logging::create(DEBUG, "main", std::move(configs));

    // Create and register before spawning
    auto logger = Logger::new_ext(DEBUG, "worker", true, true);
    logging->add_logger(*logger);

    std::thread worker([lg = std::move(logger)]() mutable {
        lg->info("hello from worker");
    });

    logging->info("hello from main");
    worker.join();
    logging->shutdown(false);
    return 0;
}
```

> `add_logger` must be called *before* the thread is spawned because it wires
> up the channel sender.  The `rust::Box<Logger>` is then moved into the
> thread's closure.
